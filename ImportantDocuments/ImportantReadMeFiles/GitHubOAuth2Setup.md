# 1. Introduction

Many modern applications allow users to login using their **GitHub account**.

Example:

```
Login with GitHub
```

This functionality works using **OAuth2 authentication**.

To enable this feature, you must create:

* **Client ID**
* **Client Secret**

These credentials are generated by creating a **GitHub OAuth Application**.

---

# 2. What is OAuth2

OAuth2 is an **authorization framework** that allows applications to access user information **without sharing the user’s password**.

Example flow:

```
User clicks "Login with GitHub"
        ↓
GitHub authentication page opens
        ↓
User grants permission
        ↓
Application receives access token
        ↓
Application retrieves user information
```

---

# 3. Why GitHub OAuth is Used

| Benefit                | Description                  |
| ---------------------- | ---------------------------- |
| Secure login           | Users do not share passwords |
| Trusted authentication | GitHub verifies identity     |
| Easy integration       | Works with APIs              |
| Industry standard      | Used by many applications    |

---

# 4. What are Client ID and Client Secret

| Credential    | Purpose                     |
| ------------- | --------------------------- |
| Client ID     | Identifies your application |
| Client Secret | Verifies your application   |

Example:

```
Client ID
Iv1.1234567890abcdef

Client Secret
1234567890abcdef1234567890abcdef
```

These credentials are required to connect your application with GitHub authentication.

# 6. Step-by-Step Guide to Create GitHub OAuth App

Process overview:

```
1. Login to GitHub
2. Open Developer Settings
3. Create OAuth Application
4. Configure application details
5. Generate Client Secret
6. Copy Client ID and Client Secret
```

---

# 7. Step 1 – Login to GitHub

1. Open your browser.
2. Go to:

```
https://github.com
```

3. Login using your **GitHub account**.

After login, you will see the GitHub dashboard.

---

# 8. Step 2 – Open Developer Settings

1. Click your **profile picture** in the top-right corner.
2. Click:

```
Settings
```

3. Scroll down the left sidebar.
4. Click:

```
Developer Settings
```

You will now see developer configuration options.

---

# 9. Step 3 – Create a New OAuth App

1. Inside **Developer Settings**, click:

```
OAuth Apps
```

2. Click:

```
New OAuth App
```

You will now see the OAuth application registration form.

---

# 10. Step 4 – Configure OAuth Application

Fill the application details.

Example configuration:

| Field                      | Example                                                                                          |
| -------------------------- | ------------------------------------------------------------------------------------------------ |
| Application Name           | Spring Security GitHub OAuth                                                                     |
| Homepage URL               | [http://localhost:8080](http://localhost:8080)                                                   |
| Application Description    | GitHub OAuth login testing                                                                       |
| Authorization Callback URL | [http://localhost:8080/login/oauth2/code/github](http://localhost:8080/login/oauth2/code/github) |

Example form values:

```
Application Name:
SpringSecurityGithubOAuth

Homepage URL:
http://localhost:8080

Authorization Callback URL:
http://localhost:8080/login/oauth2/code/github
```

Click:

```
Register Application
```

Your OAuth app will now be created.

---

# 11. Step 5 – Generate Client Secret

After creating the application, GitHub will show:

```
Client ID
```

However, the **Client Secret is not generated automatically**.

Steps:

1. Click:

```
Generate a new client secret
```

2. GitHub will generate a secret.

Example:

```
Client Secret
abc123456789abcdef123456789abcdef
```

# 12. Step 6 – Copy Client ID and Client Secret

Example credentials:

```
Client ID

Client Secret
1234567890abcdef1234567890abcdef
```

Save these credentials securely.

These values will be used in your application configuration.

# 13. Example Spring Boot Configuration

Example `application.yml`:

```
spring:
  security:
    oauth2:
      client:
        registration:
          github:
            client-id: YOUR_CLIENT_ID
            client-secret: YOUR_CLIENT_SECRET
            scope:
              - read:user
              - user:email
```

# 15. Important Security Notes

| Best Practice                   | Explanation                 |
| ------------------------------- | --------------------------- |
| Never expose Client Secret      | Keep credentials private    |
| Do not commit secrets to GitHub | Use environment variables   |
| Use HTTPS in production         | Protect tokens              |
| Restrict callback URLs          | Prevent unauthorized access |

---

# 16. Troubleshooting

## redirect_uri_mismatch

Occurs when callback URL does not match.

Solution:

Ensure callback URL matches exactly:

```
http://localhost:8080/login/oauth2/code/github
```

---

## invalid_client

Occurs when Client ID or Client Secret is incorrect.

Solution:

Check credentials.

---

## access_denied

Occurs when user denies permission.

Solution:

User must grant access.

# 17. Cleanup Guide (Delete OAuth App After Testing)

After completing testing, you may want to delete the OAuth application.

---

## Step 1 – Open Developer Settings

1. Go to:

```
https://github.com/settings/developers
```

2. Click:

```
OAuth Apps
```

---

## Step 2 – Select the OAuth Application

Find your application.

Example:

```
SpringSecurityGithubOAuth
```

Click the application name.

---

## Step 3 – Open Advanced Settings

On the application page, scroll down.

Click:

```
Advanced
```

---

## Step 4 – Delete the OAuth Application

Click:

```
Delete this OAuth application
```

GitHub will ask for confirmation.

Confirm deletion.

---

## What Happens After Deletion

| Item              | Result              |
| ----------------- | ------------------- |
| OAuth Application | Permanently deleted |
| Client ID         | Invalid             |
| Client Secret     | Invalid             |
| Access Tokens     | Revoked             |

This means the application can **no longer authenticate users via GitHub**.
